한부분의 고장이 전체 앱의 마비로 이어지지 않게 에러를 격리해야함 => 빈 화면

에러바운더리
- 하위 컴포넌트 트리에서 발생하는 렌더링 에러를 포착하고 제어하는 특수한 컴포넌트
- 클래스 컴포넌트만 가능
  - 두개의 생명주기 메소드중 하나를 구현해야함 (2개의 순서대로 호출됨)
  - getDerivedStatusFromError(error)
    - 렌더링 과정에서 에러 발생시 호출
    - 컴포넌트의 state를 갱신하는 용도
  - componentDidCatch(error, errorInfo)
    - 에러가 포착된 후 호출
    - 에러정보 + 어떤 컴포넌트가 에러를 발생시켰는지를 포함하는 stack trace

에러 전파
- 에러 발생시 이벤트와 동일하게 돔요소가 아닌 컴포넌트 트리를 따라 상위로 전파
- 에러바운더리에 포착된 에러는 더 상위로 전파 안되게 하여 격리

생명주기에서 발생하는 에러만 감지된다.
- useEffect내에서 동기적으로 포착
- 재조정과정에서 발생하는 에러만 감지
- useThrowError 훅 같은 방식으로 에러가 포착되게 할 수 있다.

- 아래의 에러는 포착되지 못함
  - 이벤트 핸들러 내부 에러
  - 비동기 fetch
  - setTimeout

react-error-boundary 패키지를 쓰던가
- 유연한 Fallback UI 렌더링
- useThrowError 비슷한거 지원
