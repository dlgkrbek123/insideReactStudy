Nodejs
- c/c++로 작성된 V8 자바스크립트 엔진을 기반으로 빌드된 JS 런타임 환경
- 싱글스레드 기반
- 이벤트 루프를 통해 여러 요청을 효율적으로 처리하는 
  async non-blocking IO 모델을 채택

- 메인 스레드의 흐름이 막히지 않고 백그라운드에 위임된다.
- await 만나면 Promise 리턴하며 콜스택에서 잠시 사라짐
  - 처리되면 나머지 부분이 다시 마이크로 태스크 큐로 삽입

V8 자바스크립트 엔진: js코드를 기계어로 컴파일하고 실행
Node API: js와 C/C++로 작성된 내부 모듈을 연결하는 다리

heap: 동적으로 생성된 객체/배열/함수가 저장되는 메모리 공간
call stack: 함수호출을 처리하는 자료구조, 현재 코드흐름 위치 추적

Libuv
- 비동기 IO 작업을 처리하는 라이브러리
- 논블로킹 동작을 지원
- 이벤트 루프 / 이벤트 큐 / 스레드 풀을 관리
- 파일시스템/DNS/네트워킁같은 비동기 작업을 처리

이벤트 큐
- 비동기 작업이 완료된 후 실행되어야할 콜백함수가 대기하는 공간
- microtask queue(Promise, await) / macrotask queue (setTimeout)
- 마이크로 태스크 큐가 우선순위가 더 높다. 

이벤트 루프
- 콜 스택이 비어있을때 이벤트 큐를 확인하여 대기중인 콜백함수를 
  콜 스택으로 삽입하여 실행하는 것을 반복하는 구성 요소

쓰레드 풀
- 요청된 비동기 작업이 수행되는 워커 스레드를 관리하는 장소
- 비동기 작업을 위임받음
  - 메인 스레드는 블로킹되지 않고 응답성을 유지
  - 논블로킹 I/O를 수행

  - nvm use

패키지 : 프로젝트에서 사용하는 라이브러리 / 모듈

package.json에 사용할 라이브러리와 버전을 명시
lock파일은 현재 프로젝트에 설치된 모든 패키지의 정확한 버전과 하위의존성이 기록

npm은 노드에서 기본적으로 제공하는 패키지 매니저 (성능이 구림)

yarn
- npm의 문제를 개선 (페이스북에서 만듦)
- 병렬 설치를 지원
- node_modules를 생성하지 않고 의존성 그래프를 관리하여 호환성 문제 있을지도

pnpm (Performant node package manager)
- 스토어라는 공용장소에 라이브러리르 설치
- 같은 버전을 여러 프로젝트에 쓴다면 공유
- 심볼릭 링크 기반
