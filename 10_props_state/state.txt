단방향 데이터 흐름
자동 배칭

------------------------------------------------------

지역상태
- 컴포넌트 내부에 선언되어 관리되는 상태
- 해당 컴포넌트의 생명주기에 귀속된다.

파생상태
- props나 다른 상태로부터 계산된 파생값
- 매번 계산하거나 useMemo()

------------------------------------------------------

state는 특정 시점의 스냅샷이다.
- setState하면 새로운 값으로 렌더링을 예약하는것일뿐 즉시 바뀌지는 않는다.
- 이벤트 핸들러가 과거의값을 기억하는 현상이 일어나곤 한다. (stale closure)

리액트는 객체의 참조를 비교하여 상태의 변화 여부를 판단하므로 
상태를 불변성을 지키면서 변경해야한다.
- useEffect의 deps같은 것도 같은 원리
- 리렌더링이 안될 수있다.
- 상태가 복잡하면 불변성 지키는게 빡세니까 immer를 써봐라

------------------------------------------------------

상태 끌어올리기
- 여러 컴포넌트가 동일한 상태를 공유하고 동기화해야할떄 
  해당 상태를 가장 가까운 조상 컴포넌트로 옮겨 관리하는 패턴
- 단일출처원칙 + 데이터 일관성

------------------------------------------------------

리액트는 부모에서 자식으로 단방향 데이터 흐름을 가진다
- 부모가 props를 내려주고 자식이 활용
- 상태 변화의 원인 추적과 예측이 수월
- 관심사가 분리

단방향 바인딩
- 모델이 뷰를 일방적으로 결정
- 뷰의 변경이 모델에 직접 영향을 주지 않는 원칙
- 뷰의 변경을 모델에 반영하려면 명시적인 요청이 필요

------------------------------------------------------

배칭
- 리액트 내부의 성능 최적화 기법
- 여러개의 상태 업데이트를 하나의 묶음(batch)로 만들어 한번의 리렌더링만 수행
- 같은 이벤트 흐름내 발생한 setState를 배치처리

flushSync
- 상태 업데이트를 DOM에 동기적으로 반영
- 자동 배칭의 동작을 의도적으로 건너뛰기

------------------------------------------------------

제어 컴포넌트
- 자신의 상태를 가지지 않고 props에 의해 완전히 제어되는 컴포넌트
